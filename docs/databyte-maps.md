# DataByteSprites Maps

Start here before changing Data Discovery files.

## Maps

- `docs/databyte-discovery-scanner-map.md` — active Scanner OS files and runtime ownership.
- `docs/databyte-battle-rebuild-map.md` — battle system notes, active files, and next steps.
- `docs/databyte-legacy-app-rebuild-map.md` — legacy app migration status.

## Rule

For each change:

1. Read the map.
2. Change the code.
3. Wire the loader if needed.
4. Update the map.
5. Report the commit and test path.
