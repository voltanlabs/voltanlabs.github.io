# Repository Inspector

## When To Use

Before auditing, debugging, refactoring, or modifying code.

## Procedure

1. Identify repository root.
2. List top level files.
3. Locate relevant files.
4. Read existing implementation.
5. Quote evidence.
6. Build file dependency map.
7. Propose changes only after inspection.

## Never

- Assume file contents.
- Assume architecture.
- Claim a fix without reading code.
