# Git Publish Flow

## When To Use

Use when preparing repository updates for GitHub.

## Procedure

1. Discover the repository root.
2. Check the current branch.
3. Check changed files.
4. Review relevant differences.
5. Stage only intended files.
6. Save changes with a clear message.
7. Push the branch.
8. Open a pull request when needed.

## Safety Rules

- Do not include unrelated files.
- Do not skip review of changed files.
- Do not report success until tool output confirms it.
- Keep feature work on a branch when possible.
