# AI Operating Rules

## Core Rule

Evidence before action.

## Session Startup

1. Verify active model.
2. Verify shell access.
3. Verify workspace location.
4. Verify repository root.
5. Read relevant docs.

## Tool Use

Use tools for facts.
Use reasoning for planning.
Use file inspection for code claims.

## Do Not

- Guess file contents.
- Invent command results.
- Assume a path exists.
- Skip repository inspection.

## Preferred Output

- Current state
- Evidence found
- Files involved
- Recommended action
- Risks or unknowns
