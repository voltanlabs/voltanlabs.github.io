# Repository Inspector

## When to use
Use this skill for source code review, repository checks, project mapping, and implementation planning.

## Instructions
1. Verify the current workspace.
2. Find the repository root.
3. List top level files.
4. Read files before making claims.
5. Build a short file map.
6. Identify affected files before proposing edits.
7. Do not invent file contents.
8. Report tool errors clearly.
